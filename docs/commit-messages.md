# Commit Message Format

These guidelines apply to all developers and AI agents contributing to **Talis**, a Next.js app for Pen-and-Paper dice rollers.

All commit messages must follow the emoji-prefixed format below to maintain consistency and improve readability of the project history.

## Commit Message Format

Use the following emoji prefixes based on the type of change:

* ✨ **New features** - Use for any new functionality or features
* 🐛 **Bug fixes** - Use for fixing bugs
* 🔧 **Configuration/refactoring** - Use for code refactoring, configuration changes, or structural improvements
* 🎨 **Styling/UI changes** - Use for visual changes, styling updates, or UI improvements
* 📚 **Documentation** - Use for documentation updates
* 🤖 **Agent/AI related changes** - Use for AI agent or automation-related changes
* 🔥 **Removal/deletion** - Use when removing code or features

## Examples

```
✨ Add dark mode support for dice rollers
🐛 Fix shadowrun dice calculation error
🔧 Refactor store utilities for better type safety
🎨 Update button styling to match new design system
📚 Add API documentation for dice roll endpoints
🤖 Update cursor rules for commit messages
🔥 Remove deprecated theme configuration
```

## Best Practices

1. **Be descriptive**: The commit message should clearly describe what changed
2. **Use present tense**: Write as if the commit is doing the action (e.g., "Add feature" not "Added feature")
3. **Keep it concise**: Aim for clarity without unnecessary verbosity
4. **One change per commit**: Each commit should represent a single logical change
